01 · Развёртывание

Восстановление, хостинг и HTTPS

Восстановление сайта из резервной копии, размещение на сертифицированном хостинге и настройка защищённого соединения.

1

Резервная копия → архив

Полный дамп БД (mysqldump) + архив файлов сайта (tar.gz). Хранение: локально + S3-совместимое облако.

2

Выбор хостинга

VPS на российской площадке (RU-CENTER, Timeweb, Selectel) — требование локализации данных по 152-ФЗ. Минимум: 2 vCPU, 4 ГБ RAM, Ubuntu 22.04 LTS.

3

Развёртывание

Nginx + PHP-FPM (или Node), настройка виртуального хоста, перенос файлов, импорт БД, проверка работоспособности.

4

HTTPS / TLS

Сертификат Let's Encrypt (certbot), редирект HTTP→HTTPS, заголовки HSTS, OCSP Stapling. Оценка SSL Labs: A+.

02 · Резервирование

Резервное копирование БД СКУД

Автоматизированное резервирование базы данных системы контроля и управления доступом (СКУД) с проверкой целостности.

🗄️

Расписание

Ежедневный полный дамп в 02:00 (cron). Инкрементальные — каждые 6 часов. Хранение: 30 дней ротация.

📦

Хранение

Локально на сервере + реплика в изолированном S3-бакете. Шифрование архивов AES-256 перед выгрузкой.

✅

Проверка

Автоматический тест восстановления раз в неделю: разворачивание в sandbox-окружении и проверка контрольных таблиц.

Скрипт резервирования (пример)

#!/bin/bash
DATE=$(date +%Y%m%d_%H%M)
DB="skud_db"
DEST="/backups/skud"

mysqldump -u root -p$PASS $DB \
  | gzip > $DEST/${DB}_${DATE}.sql.gz

# Шифрование
openssl enc -aes-256-cbc \
  -in $DEST/${DB}_${DATE}.sql.gz \
  -out $DEST/${DB}_${DATE}.enc \
  -pass file:/etc/backup.key

# Выгрузка в S3
aws s3 cp $DEST/${DB}_${DATE}.enc \
  s3://mvd-backup/skud/ --sse AES256

# Удаление файлов старше 30 дней
find $DEST -mtime +30 -delete
03 · Аналитика

Анализ эффективности и сбор статистики

Мониторинг работы платформы и обращений граждан без передачи персональных данных на зарубежные серверы.

📊

Веб-аналитика

Matomo (self-hosted) — полный контроль данных. Метрики: посещаемость, воронка подачи заявлений, отказы.

⚡

Uptime-мониторинг

Uptime Kuma или Zabbix. Алерты в Telegram при недоступности. SLA-отчёты ежемесячно.

🔍

Логи СКУД

ELK-стек (Elasticsearch + Kibana) для анализа событий доступа: пиковые нагрузки, аномалии, отчётность.

📋

KPI-дашборд

Количество заявлений в сутки, среднее время обработки, доля повторных обращений. Выгрузка в Excel/PDF.

04 · Юзабилити

Корпоративный стиль по книге «Не заставляйте меня думать»

Принципы Стива Круга применительно к сайту государственного органа: простота, предсказуемость, минимум усилий.

→

Ясная навигация с хлебными крошками

Пользователь всегда понимает, где находится и как вернуться. Один уровень навигации для ключевых задач.

→

Сканируемость вместо чтения

Заголовки, короткие абзацы, маркированные списки. Важное — крупно и вверху. Никакого «полотна» текста.

→

Очевидные CTA

«Подать заявление», «Проверить статус» — крупные кнопки с предсказуемым действием. Нет двусмысленных ярлыков.

→

Минимум шагов в форме

Шаги нумерованы и видны сразу (прогресс-бар). Поля — только необходимые. Автозаполнение где возможно.

→

Ошибки на русском, с решением

«Введите серию паспорта (4 цифры)» вместо «Неверный формат». Каждое сообщение об ошибке — инструкция.

06 · Лендинг

Маркетинговый лендинг (Webflow / Framer)

Одностраничный сайт для первичного информирования. Выбор инструмента: Webflow (больше контроля над CMS) или Framer (быстрее прототипирование).

Структура лендинга

1Hero: кому нужна регистрация и зачем
2Пошаговый процесс (3 шага)
3Документы (список + PDF)
4FAQ (аккордеон)
5Запись на приём / кнопка Госуслуг

Webflow vs Framer

Webflow Встроенная CMS, подходит для частого обновления контента (графики, документы)
Framer Богатая анимация, быстрый старт, меньше возможностей CMS
→ Выбор Webflow — из-за частого обновления расписания и документов

Почему Astro

Частичная гидратация (Islands): статические страницы с мгновенной загрузкой + интерактивные виджеты там, где нужно. Lighthouse ≥ 95.

09 · Соответствие и архитектура

152-ФЗ и архитектурная документация

Работа с персональными данными иностранных граждан требует строгого соблюдения Федерального закона «О персональных данных».

Требования 152-ФЗ

  • Серверы и БД — только в РФ (ст. 18.1)
  • Согласие на обработку ПДн при регистрации
  • Политика конфиденциальности на сайте
  • Разграничение прав доступа к ПДн
  • Журнал доступа и изменений ПДн
  • Срок хранения — не дольше цели обработки
  • Уведомление Роскомнадзора об обработке ПДн
  • Шифрование при передаче (TLS 1.2+)
